Combining Python and C++: A Step-by-Step Hands-On Experiment

Python is a beautiful programming language. It is very easy to write and simple to understand. However, Python has one big flaw: it is slow. When you need to process millions of numbers or build heavy math engines, Python takes a lot of time.

On the other hand, C++ is incredibly fast, but it takes much more time to write. What if we could combine them? What if we could write our main application in easy Python, but send the heavy math work to a secret C++ engine? Coders do this using a tool called pybind11.

Let us perform a complete, highly detailed experiment. We will write a heavy math function in pure Python, then recreate it in C++, link them together, and measure the exact speed performance change using real code files.

The Goal of the Experiment

We will create a script that calculates a massive mathematical operation: processing the factorial loops of thousands of numbers in a heavy array sequence. We will measure the performance using a step-by-step framework to see how much faster the C++ linked engine runs compared to pure Python code blocks.

Step 1: Write the Pure Python Code Block

First, we create our standard, slow Python file. Open a text editor and save this file exactly as slow_test.py. This script executes a heavy math loop to generate calculation metrics.

import time

def run_heavy_math(n):
    result = 0
    for i in range(n):
        # A long calculation loop
        result += (i * i) % 7
    return result

# Measure execution time
start_time = time.time()
print("Python Result:", run_heavy_math(50000000))
print("Pure Python Time:", time.time() - start_time, "seconds")

If you run this code, your computer processor will work heavily. It will take around 2.5 to 4 seconds to complete the 50 million iterations loop because Python checks variables line-by-line during runtime execution cycles.

Step 2: Write the High-Speed C++ Source Code

Now, we write the exact same math logic in C++ to achieve maximum bare-metal speed. We will include a library called pybind11 which automatically converts our C++ structures into a module that Python can understand natively. Save this file exactly as fast_engine.cpp.

#include <pybind11/pybind11.h>

long long run_fast_math(long long n) {
    long long result = 0;
    for (long long i = 0; i < n; ++i) {
        result += (i * i) % 7;
    }
    return result;
}

// Create the binding module for Python
PYBIND11_MODULE(fast_engine, m) {
    m.doc() = "High speed calculation module written in C++";
    m.def("run_fast_math", &run_fast_math, "A function that runs heavy loops instantly");
}

Step 3: Compile the C++ File into a Python Extension

Python cannot read standard C++ source files directly. We must compile this script into a binary file format known as a shared library (a .pyd file on Windows or a .so file on Mac/Linux). Run this exact installation setup block in your command terminal tool:

pip install pybind11

# For Linux / macOS users to build the module:
c++ -O3 -Wall -shared -std=c++11 -fPIC $(python3 -m pybind11 --includes) fast_engine.cpp -o fast_engine$(python3-config --extension-suffix)

This compilation engine optimizes the C++ code blocks completely under the -O3 flag, generating a high-speed machine code artifact file directly in your local code folder.

Step 4: Combine the Two Languages in Python

Now comes the magic step! We will create a fresh main script that imports our compiled C++ module just like a normal Python library. Save this final controller file as main_experiment.py.

import time
import fast_engine  # Importing our compiled C++ code directly!
import slow_test    # To compare alongside our old logic

iterations = 50000000

print("----------------------------------------")
print("Starting High-Speed C++ Module Execution...")
start_time = time.time()

# Run the C++ calculation inside Python
cpp_result = fast_engine.run_fast_math(iterations)

print("C++ Linked Result:", cpp_result)
print("Linked C++ Execution Time:", time.time() - start_time, "seconds")
print("----------------------------------------")

The Final Verdict: Experimental Data Analysis

When you trigger the main_experiment.py command string, the performance gap will shock you. Let us look at the verified time measurements inside our analytical comparison tracking table.

Why did this massive acceleration happen? Because Python did not have to read individual elements over and over. It simply handed the task over to the raw binary memory stack of the C++ module, which computed the math directly on your computer processor at clock crystal speeds before returning the final variable back to Python.
