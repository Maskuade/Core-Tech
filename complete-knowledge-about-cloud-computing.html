What Is Cloud Computing: -

Cloud computing is the on-demand delivery of computing resources (servers, storage, databases, networking, and software) over the internet on a pay-as-you-go basis.

Instead of buying and owning your own physical servers and data centers, you rent computing power from a cloud provider such as Amazon (AWS), Microsoft (Azure), or Google (GCP).

When it was introduced:

The concept of modern cloud computing was born in 2006 when Amazon launched AWS EC2 (Elastic Compute Cloud), which allowed anyone in the world to rent virtual servers over the internet on a pay-as-you-go basis.

What was the main goal of introducing it: -

The primary goal of cloud computing is to eliminate the requirement for businesses to purchase and manage physical hardware. In the old days, companies had to drop big bucks on servers and storage, often throwing money away by buying more than they needed.

Cloud computing changes this by allowing companies only to pay for what they use, to scale up or down as needed, and to access computing power quickly. "Basically, it means computing can be as flexible and cheap as electricity. You pay for what you use, and you don't own the infrastructure

Who introduced it and why and where: -

Cloud computing was introduced by Amazon in 2006 with AWS EC2, utilizing their underused infrastructure to rent computing power to businesses. This innovation turned excess capacity into profit.

Before Amazon, Salesforce launched the Software as a Service (SaaS) model in 1999. However, Amazon's EC2 was the first true infrastructure as a service (IaaS), allowing users to rent virtual servers online. Consequently, Amazon is seen as the father of modern cloud computing, while Salesforce is recognized as the pioneer of cloud software.

Cloud Computing Characteristics:

On-Demand Self-Service:-

On-demand self-service is a fundamental aspect of cloud computing, allowing users to provision resources automatically without a provider assistance. Users can access a web console, utilize an API, or run command-line scripts to create virtual servers, allocate storage, set up databases, or configure networking—all independently and within minutes.

In contrast, traditional on-premises systems involved a slow process of requests, approvals, hardware orders, and delays for installation. Cloud computing eliminates these hurdles.

Whether you need one server or many, the cloud provides instant control. This self-service feature enhances agility—developers can quickly create test environments, businesses can launch products swiftly, and everyone can begin without waiting for others.

Broad Network Access:

Resources are available over the network and accessed through standard mechanisms that promote use by heterogeneous thin or thick client platforms (e.g., mobile phones, laptops, and workstations).

Unlike traditional systems that require physical connections, cloud services are accessed through web browsers, mobile apps, command-line tools, and APIs. This means developers can deploy servers from anywhere, salespeople can see customer info on the road, and engineers can monitor systems remotely.

Different operating systems are not a problem, thanks to cloud providers. Real-time data access makes remote work and global collaboration possible. Cloud computing also increases availability quite a lot. Without such wide availability, cloud computing would be limited to certain devices.

Resource Pooling:-

The cloud provider uses a multi-tenant model, where computing resources are pooled and shared among multiple customers. Rather than dedicating a physical server to a single customer, they pool computing power, storage, and networking from thousands of servers within huge data centers. Resources are allocated as needed, often without the customer knowing what specific physical machine they are using.

This method achieves economies of scale, allowing a single server to host numerous virtual machines from various companies, thereby lowering costs. Providers handle load balancing, fault tolerance, and resource allocation so customers only use what they need. Resource pooling is crucial for the cost-effectiveness of cloud computing, as it allows customers to share a large resource pool and efficiently meet demand.

Rapid Elasticity:-

Rapid elasticity, or scalability, is a key cloud feature that enables quick and often automatic adjustment of computing resources to match demand. During traffic spikes, such as flash sales or product launches, the cloud can swiftly add servers and increase bandwidth without manual effort. When traffic decreases, it automatically reduces resources, saving costs. Unlike traditional on-premises systems, which require advance planning and leave you with unused hardware, cloud elasticity allows for rapid scaling from one to thousands of servers.

This process is frequently automated through tools like auto-scaling groups that adjust resources based on metrics like CPU usage. Ultimately, this ensures optimal capacity management, providing sufficient power during peak times while minimizing waste during low traffic periods.

Measured Service:-

Measured service, or the pay-as-you-go model, allows customers to pay only for the cloud computing resources they actually use. Cloud providers monitor and report usage, tracking every action—such as server runtime, data storage, and API calls—resulting in precise billing. This contrasts with traditional IT, where companies pay upfront for hardware regardless of usage. With measured service, if you use a server for 3 hours, you pay for 3 hours.

This model applies to all service types: IaaS, PaaS, SaaS, and FaaS. Cloud providers offer clear dashboards and billing reports for cost transparency, enabling budget predictability. This shifts IT costs from capital expenditure (CapEx) to operational expenditure (OpEx), ensuring you only pay when resources provide value.

Cloud Deployment Models:-

Cloud deployment models specify the geographical location of cloud infrastructure, the management responsibilities, and access permissions. They encompass four main types, each tailored to different organizational needs and operational strategies, ensuring flexibility and control over cloud resources.

Public Cloud:-

Public cloud refers to a model where a third-party provider manages cloud infrastructure, making it available to the public online. Customers share the same physical resources in a multi-tenant architecture. Major providers include AWS, Microsoft Azure, GCP, Alibaba Cloud, and IBM Cloud, which handle everything from server procurement to security and maintenance.

Users can easily sign up and pay only for what they use, making it suitable for startups and businesses with fluctuating demands. While it offers significant scalability and cost-effectiveness, it also presents challenges like limited visibility into infrastructure and reliance on the provider's security measures. Overall, the public cloud is often the preferred option due to its scalability and global accessibility.

Private Cloud:-

Private cloud is a dedicated cloud infrastructure for a single organization, providing benefits like scalability, self-service, and resource pooling in an isolated environment. Unlike public clouds, which share resources among multiple users, private clouds are exclusively used by one organization, either on-premises or managed by a third party.

This model offers enhanced control, security, and compliance, allowing customization of networking, security policies, and hardware. It is favored in regulated industries such as government, finance, and healthcare. However, it requires significant investment in hardware and in-house expertise for management.

Essentially, a private cloud offers the advantages of cloud computing within the organization’s own infrastructure, but it demands a substantial budget and skilled team to maintain.

Hybrid Cloud:

Hybrid cloud integrates private and public clouds for efficient data sharing. It enhances security and compliance for sensitive workloads while providing the scalability and cost benefits of public clouds. For example, banks may keep customer data on private clouds for regulations but use public clouds during peak times.

This model is also useful for disaster recovery and real-time data replication. Common applications include developing and testing in public clouds before moving to private ones and performing big data analytics while keeping raw data secure on-premises. A strong network connection, often through VPNs or dedicated links, is essential for hybrid cloud operations, making it ideal for enterprises with substantial on-premises investments transitioning to public cloud solutions.

Multi-cloud:

Multi-cloud is a strategy where organizations utilize two or more public cloud providers simultaneously, distributing workloads to avoid reliance on a single vendor. This differs from hybrid cloud, which combines private and public clouds. Companies may run applications on AWS, store data on Google Cloud, leverage Azure for AI, and manage disaster recovery on IBM Cloud.

Key reasons for adopting multi-cloud include the following:

  • Avoiding vendor lock-in:

  • Spreading workloads allows for better pricing and flexibility to switch providers without major overhauls.

  • Best-of-breed services:

  • Organizations can select the best offerings from each provider, such as AWS Lambda for serverless computing and Google BigQuery for data warehousing.

  • Redundancy and resilience:

  • If one provider fails, workloads on others remain operational, enhancing business continuity.
    However, multi-cloud can be complex, requiring expertise across platforms, management of various billing and security systems, and effective networking tools to ensure seamless integration.
    This strategy is most common among large enterprises with the budget and technical staff to handle the complexity, while smaller organizations typically start with a single public cloud provider and evolve to multi-cloud only when specific business needs arise.

Cloud Computing Service Models: -

Cloud computing service models describe how providers allocate resources to customers, balancing management responsibilities between the customer and the provider. These models vary from high customer control to high provider management.

The primary aim is flexibility, catering to diverse customer needs. For example, large enterprises may seek complete control, while startups might prefer full provider management to concentrate on their core activities. Developers may only need to run simple span without server management.

Key service models include the following:

  • Infrastructure as a Service (IaaS)

  • Platform as a Service (PaaS)

  • Software as a Service (SaaS)

  • Platform as a Service (PaaS)

Each model defines the responsibilities of the provider and the customer, influencing costs. Each model clarifies the roles of the provider and customer, affecting costs, scalability, security, and management. Choosing the right model is crucial to avoid unnecessary expenses or loss of control.

Infrastructure As A Service (IAAS):

Infrastructure as a Service (IaaS) is a cloud computing model that provides on-demand access to virtualized resources such as servers, storage, and networking over the internet. The cloud provider manages the physical infrastructure, while customers control the operating systems and applications, offering significant flexibility and customization.

Notable IaaS solutions include Amazon EC2 for virtual servers and Amazon S3 for storage. Benefits include reduced hardware costs, instant scalability, and full control of the computing environment, though customers are responsible for updates, security, and backups.

IaaS is ideal for IT-savvy organizations needing cloud flexibility while maintaining control, similar to renting an apartment where the landlord provides the space, but you personalize it.

Platform As A Service (PAAS):

Platform as a Service (PaaS) is a cloud computing model that provides a complete environment for application development and deployment, managing everything from operating systems to databases. This enables developers to focus on their applications without the burden of infrastructure management.

Key benefits of PaaS:

  • Enhanced developer productivity (no server setup required)
  • Quick deployment (launch applications in minutes)
  • Automatic scalability (platform adjusts to traffic demands)
  • Lower operational costs (no dedicated infrastructure team needed)

However, PaaS has limitations: users are confined to the languages and tools offered by the provider and have less control compared to Infrastructure as a Service (IaaS). It is ideal for teams building web apps, mobile backends, APIs, and microservices, allowing them to focus on coding rather than infrastructure management.

In summary, PaaS is akin to renting a fully furnished apartment where the landlord handles everything—you simply provide your application span and start using it immediately.

Software As A Service (SAAS):

Software as a Service (SaaS) is a cloud model delivering software applications via the internet, managed by the provider. Customers access it through a web browser or mobile app without installation.

They pay a subscription fee based on usage or users and can start immediately. Examples include Google Workspace, Microsoft 365, and Salesforce.

Benefits of SaaS:

  • No infrastructure management
  • Automatic updates
  • Accessible from any internet-enabled device
  • Predictable pricing
  • Quick deployment

Trade-offs include limited customization, potential data security concerns, and reliance on internet connectivity. SaaS suits businesses needing standard applications like email and collaboration tools without software management hassles.

In summary, SaaS is like a hotel stay: ready to use without maintenance worries.

### Function As A Service (FAAS) / Serverless:-

Function as a Service (FaaS), or Serverless computing, is a cloud model enabling developers to execute span—functions—in response to events without managing servers. The cloud provider manages everything, including server setup, maintenance, and scaling. Developers write functions in languages like Python or Java, upload them, and set triggers like HTTP requests. Payment is based on actual compute time, not idle resources.

Key benefits of FaaS:
  • Cost efficiency: No charges for idle servers.
  • Automatic scalability: Functions run in parallel as needed.
  • Low operational overhead: No server management required.
  • Event-driven architecture: Suits modern applications.

Trade-offs include:
  • Cold start latency: Initial execution may be slower if the function hasn't run recently.
  • Execution time limits: Functions usually run for 5-15 minutes.
  • Statelessness: Functions can't retain data between executions.

FaaS is ideal for tasks like image processing and API backends, akin to ordering room service: you pay only for what you use.

Cloud Storage Types:-

Cloud storage types are methods for organizing and accessing data in the cloud, each designed for specific needs with trade-offs in performance, scalability, cost, and accessibility. Different data types require tailored storage solutions. Each type serves distinct functions, impacting your application's performance, reliability, and costs.

Direct Attached Storage: -

Direct Attached Storage (DAS) connects hard drives (HDDs) or solid-state drives (SSDs) directly to a single server or virtual machine, bypassing network use. In cloud computing, DAS serves as local storage for a virtual machine, referred to as an instance store or ephemeral storage, enabling rapid access and high input/output operations per second (IOPS).

However, DAS is limited to one virtual machine, restricting access from other servers. Data stored on DAS is often temporary; if the virtual machine is stopped or crashes, the data is lost. DAS is ideal for temporary tasks like caches, scratch data, and log files that can be recreated.

Cloud providers usually offer DAS as "instance store" volumes with specific instance types, typically at no extra cost. Think of DAS as a glove compartment in a car—it's easily accessible but loses everything if the car is irreparably damaged.

File Storage (NAS):

Network Attached Storage (NAS) provides hierarchical organization of data that can be accessed simultaneously by multiple users or servers over a network. NAS is a shared drive, unlike Direct Attached Storage (DAS), which is a single server. NAS uses protocols like NFS and SMB. It supports the file management operations like create, read, update, and delete and hence is suitable for a collaborative environment.  
Cloud file storage options provide scalable storage with redundancy. NAS: Think of a shared locker in an apartment. If you have permission, you can get to it. Slower than DAS because you have to go through the network.

Block Storage:-

Block storage breaks data into fixed-size pieces that virtual machines can access like a raw hard drive. Unlike file storage, which organizes data in folders, block storage looks like an empty disk to the operating system and needs a file system (like ext4 or NTFS). Each piece works on its own and can be spread across different physical devices, but the virtual machine sees them as one unit.

This setup provides high speed with low delay and quick input/output operations, allowing multiple disks to read and write at the same time. Block storage is great for databases, business applications, virtual machine boot disks, and tasks needing steady low-latency performance. In cloud environments, it often acts as persistent volumes that stay intact even when the virtual machine is off, allowing them to connect to other machines.

To put it simply, if DAS is like a glove compartment and file storage is a shared locker, block storage is like a piece of land you can build on, linked to your house (the virtual machine) by a fast private road. Access is private unless shared, and this land can be moved between houses if needed. Unlike file storage, which limits you to set shelves, block storage gives you a flexible solution for better performance.

Object Storage:-

Object storage manages data as individual units called objects, which consist of the data, metadata, and a unique identifier. Unlike traditional file storage, it employs a flat address space for better scalability. Objects are stored in buckets and accessed via API calls using unique keys, regardless of their physical location. This format is perfect for unstructured data like photos and videos, where entire objects are replaced instead of modified.

Benefits include unlimited scalability, high durability (up to 99.999999999%), built-in versioning, lifecycle management, and cost-effectiveness for infrequently accessed data.

Object storage resembles a warehouse of labeled boxes. You retrieve a box by its label, pay only for what you store, and the warehouse can expand without reorganization. However, it has higher latency than block storage and is unsuitable for databases or active transactions, making it ideal for modern cloud data needs.

Content Delivery Network (CDN):

A Content Delivery Network (CDN) is a network of servers located worldwide that stores copies of your content—like images, videos, and web pages—and delivers them to users from the nearest server. This reduces the time it takes for users to access content. For instance, a user in Tokyo can load an image much faster from a local CDN server than from a distant server in Virginia.

CDNs are optimized for quickly distributing static content and work best alongside an origin storage source. Benefits include faster content delivery, less strain on your main servers, protection against DDoS attacks, lower bandwidth costs, automatic failover, and an enhanced user experience.

Think of a CDN as placing vending machines with popular items in neighborhoods worldwide, allowing users to get what they want instantly. However, CDNs are ideal for static content and may have a slight delay when new content is first published as it spreads to all servers. CDNs are crucial for global applications, streaming services, e-commerce sites, gaming, and software distribution.

Architecture:-

Cloud computing architecture consists of various components that collaborate to provide cloud services. These components are categorized into three main areas: the front end (user interface), the back end (provider infrastructure), and the delivery network (connectivity).

Front End:-

The front end of cloud computing architecture is the user interface that enables clients to access cloud services through various applications. It includes two main types of clients:

  • Thin clients: Lightweight devices or web browsers that depend entirely on the cloud for processing and storage, allowing access from any device without software installation.
  • Fat clients: Fully featured applications installed locally that utilize cloud services for enhanced functionality, offering a better user experience with offline capabilities.
    Additionally, the front end encompasses user interface elements like dashboards and APIs, which allow users to manage resources, monitor usage, and control services efficiently.
    Users interact with the front end when uploading files, launching virtual servers, or generating reports. It serves as the interface that connects them to the back end, facilitating communication through requests and responses, bridging the gap between users and the underlying cloud infrastructure.

Back-end:

The back-end of cloud computing architecture is essential for supporting user interactions through the front end. It includes both physical and virtual resources, such as processing servers, data storage systems, and management software.

When a virtual machine is activated, it runs on a physical server in the provider's data center, with files stored in back-end storage arrays. These resources are organized into global clusters and data centers, acting as a unified pool. The back-end also incorporates hypervisors for virtualization, load balancers for traffic management, and platforms for authentication, billing, and monitoring.

It is standardized, automated, and designed to be fault-tolerant, ensuring seamless service continuity in case of component failures. While invisible to users, the back-end is crucial; without it, the front end lacks functionality.

Cloud-based Delivery & Network:-

Cloud-based delivery and networks are the infrastructure that connects users to cloud resources, facilitating data exchange. It operates over three main networks:

  • Internet: Offers global access to cloud services from any internet-enabled device, commonly used for public clouds.
  • Intranets: Private networks within organizations that ensure secure communication between on-premises systems and private or hybrid clouds without public exposure.
  • Intercloud: Connects various cloud providers, enabling seamless data and service movement across platforms like AWS, Azure, and Google Cloud for multi-cloud strategies and disaster recovery.

Key components include load balancers for traffic distribution, firewalls for security, and VPNs for encrypted connections.

Virtualization:-

Virtualization is the technology that creates virtual versions of physical computing resources such as servers, storage devices, networks, and operating systems, allowing multiple virtual instances to run on a single physical machine.

Virtual Machines:-

A Virtual Machine (VM) is a software simulation of a physical computer that operates its own operating system and applications, functioning like a real machine. It exists as a set of files on physical hardware managed by a hypervisor. Each VM believes it has its own CPU, memory, storage, and network interfaces, but these resources are actually shared from the physical server.

Multiple VMs can run on a single physical machine, isolated from one another—if one VM fails or gets infected, the others remain safe. VMs are essential in Infrastructure as a Service (IaaS), enabling quick provisioning of virtual servers without needing physical hardware.

Key benefits of VMs include:

  • Portability: Move between physical servers without downtime.
  • Snapshots: Save the exact state of a VM at any time.
  • Cloning: Create copies for testing or scaling.
  • Hardware Independence: VMs work regardless of the underlying hardware.

Common uses of VMs in the cloud involve running websites, databases, and applications that typically need a physical server. In essence, a virtual machine is a simulated computer that behaves like a real one, allowing multiple VMs to operate on a single physical machine.

Hypervisor:-

A hypervisor, or virtual machine monitor (VMM), is essential software that enables multiple virtual machines (VMs) to run on a single physical server. It manages resources like CPU, memory, and storage, ensuring each VM operates independently and securely.

There are two types of hypervisors:

  • Type 1 (bare-metal): Runs directly on hardware, offering superior performance and security, commonly used in data centers and cloud services.
  • Type 2: Operates on top of an existing operating system, suitable for personal use and development.

Popular examples include VMware ESXi, Microsoft Hyper-V, KVM, and Oracle VirtualBox.

Without a hypervisor, virtualization wouldn't exist, allowing one server to host many VMs, which is fundamental to cloud computing. In essence, the hypervisor orchestrates the operation of VMs, ensuring they function smoothly without interference.

Bare Metal:-

Bare metal is a dedicated physical server for a single customer, providing direct access to all hardware resources like CPU, RAM, storage, and network interfaces without virtualization. Think of it as renting a whole house, while virtual machines are like apartments and containers are desks in a coworking space. Major cloud providers now offer bare metal servers for short-term use, enabling workload management without physical data center access.

Benefits of bare metal:

  • Maximum performance (no hypervisor overhead)
  • Predictable behavior (no virtualization issues)
  • Physical isolation for compliance (HIPAA, PCI-DSS, FedRAMP)
  • Full control over the OS and hardware
  • Simpler licensing based on physical cores

Trade-offs:
  • Slower provisioning (15-60 minutes vs. 2 minutes for VMs)
  • Less elasticity (adding capacity requires new servers)
  • Higher costs (paying for the entire server regardless of usage)
  • No snapshots or live migration (downtime during hardware failure)
  • Manual maintenance for OS updates and security
  • Manual OS updates and security maintenance

Bare metal is ideal for high-performance computing, gaming, heavy databases, compliance-focused industries, machine learning, and legacy applications. It’s less suitable for rapidly scaling workloads, small apps, short-term needs, development environments, or tight budgets.

Security:-

Cloud security is the collection of policies, technologies, and controls used to protect data, applications, and infrastructure in cloud computing environments from threats, breaches, and unauthorized access.

Data Encryption: -

Data encryption transforms readable data (plaintext) into an unreadable format (ciphertext) using algorithms and cryptographic keys. This process ensures that intercepted data remains unintelligible without the decryption key. It serves as a crucial defense in cloud security; if other protections fail and an attacker accesses your data, encryption renders it useless.

In cloud settings, encryption is applied in three states:

  • At rest: Data stored on disks and backups is encrypted automatically by cloud providers before being saved and decrypted when accessed, using keys you control.
  • In transit: Transport Layer Security (TLS) protocols protect data moving between your browser and cloud services, as well as between different cloud services.
  • In use: This involves data being processed in memory, which is less common and more complex.

Key management is vital—deciding who controls the encryption keys. You can allow the cloud provider to manage them (easier but less control), use customer-managed keys (more control but more responsibility), or bring your own keys (HYOK) stored in a hardware security module.

Without encryption, data in the cloud is exposed; anyone gaining access to the infrastructure can read it. Proper encryption safeguards your data even if physical media is stolen or lost, ensuring protection against unauthorized access.

Access Control:

Access control allows only verified users, applications, or services to access specific cloud resources based on their roles, following the principle of least privilege—granting only necessary permissions. This is managed through Identity and Access Management (IAM) systems, where user identities are created, assigned roles (like Admin, Developer, Viewer, or Billing), and linked to policies that define their allowed actions.

For example, a developer can create and delete virtual machines but cannot access production databases, while a database administrator can modify schemas but not change firewall rules. The key approach is to start users with no permissions and gradually add what they need, preventing unnecessary broad access.

In cloud environments, overly permissive IAM roles can lead to data exposure or unauthorized activities, such as cryptocurrency mining. Modern access control also considers factors like time, IP address, device compliance, and location, allowing access only during business hours from secure devices.

Regular reviews of access permissions are essential since roles can change, and permissions may remain if not revoked. Cloud providers offer tools like access analyzers to monitor permissions, anomaly detection for unusual usage, and just-in-time access requests for temporary elevated privileges. Poorly configured access control is a major cause of cloud data breaches, often due to simple errors like making storage buckets public or granting excessive permissions.

Multi-Factor Authentication (MFA):

Multi-Factor Authentication (MFA) enhances security by requiring users to provide two or more verification factors to access cloud accounts. This method combines something you know (like a password), something you have (like a smartphone or security key), and something you are (like a fingerprint), making it much harder for attackers to breach accounts, even if they obtain your password.

Relying solely on passwords is risky—people often reuse them, fall for phishing scams, and choose weak options like "password123." With MFA, even if an attacker steals your password, they cannot log in without the second factor, such as a one-time span from an authenticator app or a fingerprint.

MFA blocks over 99.9% of account compromise attempts, yet many still avoid it due to the slight inconvenience of using their phone or a key. In cloud environments, MFA is essential for privileged accounts—root users and administrators who can access sensitive data.

Common MFA methods include:

  • Time-based one-time passwords (TOTP) from apps like Google Authenticator
  • Push notifications for login approvals
  • Hardware security keys like YubiKey
  • SMS text messages (less secure due to SIM swapping risks)
  • Biometrics like fingerprints or facial recognition

Most cloud providers enforce MFA for root accounts and encourage it for all users, often using adaptive MFA that prompts for additional factors only when a login appears risky.

Disabling MFA is a critical mistake; thinking a strong password is enough is misleading. Many recent data breaches could have been avoided with MFA enabled.

In summary, MFA strengthens account security by adding multiple layers of protection. Enable MFA on every cloud account, for every user, without exception.

Shared Responsibility Model:-

The shared responsibility model is one of the most important security concepts in cloud computing that accurately defines which security activities are conducted by the cloud provider and which are left to the customer. Having a clear understanding of this division is absolutely critical because the number one reason for cloud security breaches is misunderstanding this.

Provider's Responsibility (Security of the cloud):

The cloud provider ensures the security of the cloud infrastructure, managing everything unseen, such as data centers, hardware, networks, and essential software. They implement physical security measures like 24/7 guards, biometric access, and surveillance systems.

The provider oversees all hardware, including servers and storage, ensuring proper maintenance and secure disposal of old equipment. They also protect the virtualization layer, allowing multiple customers to run virtual machines on the same server without data access issues.

Additionally, the provider safeguards the network infrastructure, preventing unauthorized access and ensuring customer isolation. They guarantee availability and redundancy by distributing data centers across various regions and maintaining Service Level Agreements (SLAs) for uptime. Compliance with certifications like SOC 1, SOC 2, ISO 27001, PCI DSS, HIPAA, FedRAMP, and GDPR is also managed, benefiting customers' regulatory needs.

In essence, if it's not visible or manageable by you, the provider secures it, allowing you to focus on your applications and data. However, once you create a virtual machine or deploy an application, you enter your own responsibility zone.

Customer's Responsibility (Security in the cloud):

The customer is responsible for security in the cloud, covering everything they create, configure, upload, deploy, or manage. Many security failures occur because customers underestimate their responsibilities or mistakenly believe the provider will handle them.

Your responsibilities begin when you launch a virtual machine:

  • Choose a secure operating system image.
  • Configure it properly and apply security patches regularly.
  • Set up host-based firewalls and install antivirus software if necessary.
  • Disable unnecessary services and ports.
  • Manage user accounts and encrypt sensitive data on the VM's disks.

For containers, ensure secure image building, vulnerability scanning, and base image updates. For serverless functions, focus on writing secure span and managing third-party dependencies.

You are fully responsible for your data—encrypting it at rest and in transit, managing encryption keys, implementing backup and disaster recovery strategies, and preventing accidental exposure due to misconfigured permissions.

Access management is your duty:
  • Create user identities and groups.
  • Assign IAM roles and policies based on least privilege.
  • Enable multi-factor authentication for all accounts, especially administrative ones.
  • Regularly review and revoke unused permissions.
  • Monitor for suspicious login attempts.

Network security is also your responsibility:
  • Configure security groups, access control lists, route tables, and web application firewalls.
  • Ensure only necessary ports are open and trusted IP addresses can access sensitive resources.

You must monitor and log activities by enabling logging services, setting alerts for unusual activity, analyzing logs for signs of compromise, and investigating security incidents.

Finally, ensure compliance with regulations like GDPR, HIPAA, or PCI DSS by implementing required controls, conducting audits, and maintaining documentation.

The cloud provider offers tools to assist, but they won't do the work for you. Misconfigurations can lead to vulnerabilities, such as public storage buckets or overly permissive IAM roles. In the shared responsibility model, the provider secures the infrastructure, but you must secure your own environment. Many customers neglect this and face consequences.

Benefits:

1) Cost Benefits

1) No upfront hardware purchases

2) No wasted money on idle servers

3) Pay-as-you-go pricing (per hour, per GB, per request)

4) No data center electricity, cooling, or real estate costs

Cloud computing is the on-demand availability of computer resources and services over the internet. Instead of having to build and maintain your own physical data centers or hard drives, you simply rent access to these capabilities from companies like Amazon Web Services or Microsoft Azure, and pay only for what you use.
